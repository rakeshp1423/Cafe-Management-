PROJECT OVERVIEW
================================================================================
TheMenu is a comprehensive, full-stack web-based café management platform built 
with Django. It's designed to streamline café operations, enhance customer 
experience, and provide an intuitive interface for managing table reservations, 
menu items, customer feedback, and user accounts. The application features a 
modern UI with warm and earthy aesthetics, interactive animations, and responsive 
design patterns.

CORE FEATURES
================================================================================
1. HOME PAGE
   - Welcoming hero section with introductory messaging
   - Sticky navigation bar for easy access to all sections
   - Social links and contact information in footer

2. MENU SYSTEM
   - Dynamic display of café items/products
   - Text overlays on product images
   - Interactive hover effects (zoom, grayscale-to-color transitions)
   - Organized menu categorization

3. RESERVATION SYSTEM
   - Seamless table booking interface for customers
   - Date, time, and party size selection
   - Reservation status tracking (pending, confirmed, completed)
   - Manager approval workflow

4. CUSTOMER FEEDBACK
   - Feedback submission form with star ratings
   - Image upload functionality for reviews
   - Public feedback display and reviews section
   - Feedback management in manager dashboard

5. USER AUTHENTICATION & PROFILES
   - Secure user registration and login
   - Two-factor authentication (2FA) support
   - User profile management
   - OTP verification for account security
   - Session tracking and user activity logs

6. MANAGER DASHBOARD
   - Administrative interface for café staff
   - Manager login with secure authentication
   - Reservation management and approval
   - Feedback moderation
   - User and booking statistics

7. VISUAL ENHANCEMENTS
   - Parallax scrolling effects
   - Image hover animations (zoom, blur, overlay effects)
   - Responsive design for mobile and desktop
   - Interactive JavaScript functionality
   - CSS animations and transitions

INSTALLATION & SETUP
================================================================================
PREREQUISITES:
  - Python 3.10+
  - pip (Python package manager)
  - Virtual environment (recommended)

STEPS:

1. Clone the repository:
   git clone https://github.com/rakeshp1423/Cafe-Management-.git
   cd TheMenu

2. Create and activate virtual environment:
   python -m venv venv
   
   On Linux/Mac:
   source venv/bin/activate
   
   On Windows:
   venv\Scripts\activate

3. Install dependencies:
   pip install -r requirements.txt

4. Run database migrations:
   python manage.py migrate

5. Create superuser (admin account):
   python manage.py createsuperuser

6. Start development server:
   python manage.py runserver

7. Access the application:
   http://127.0.0.1:8000/

8. Admin panel:
   http://127.0.0.1:8000/admin/

DEPLOYMENT TO RENDER
================================================================================
1. Push code to GitHub repository

2. Create Render account at render.com

3. Create Web Service:
   - Connect GitHub repository
   - Set Runtime: Python 3
   - Build Command:
     pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate
   - Start Command:
     gunicorn TheMenu.wsgi:application --bind 0.0.0.0:$PORT

4. Configure Environment Variables:
   - DEBUG: false
   - SECRET_KEY: [Generate at djecrety.ir]
   - ALLOWED_HOSTS: [your-app-name.onrender.com]
   - DATABASE_URL: [Auto-set by Render PostgreSQL]

5. Create PostgreSQL Database on Render:
   - Name: themenu-db
   - This auto-populates DATABASE_URL

6. Deploy:
   - Click "Create Web Service"
   - Render automatically builds and deploys
   - App available at: https://[your-app-name].onrender.com

CONFIGURATION
================================================================================
ENVIRONMENT VARIABLES (settings.py):
  - DEBUG: Enables debug mode (set to 'false' in production)
  - SECRET_KEY: Django secret key for security
  - ALLOWED_HOSTS: Comma-separated list of allowed domains
  - DATABASE_URL: PostgreSQL connection string (auto-set by Render)

DATABASE:
  - Local: SQLite (db.sqlite3)
  - Production: PostgreSQL via dj-database-url

STATIC FILES:
  - Location: /static/
  - Collected to: /staticfiles/ (production)

MEDIA FILES:
  - User uploads stored in: /media/
  - Feedback images: /media/feedback_images/

SECURITY SETTINGS:
  - CSRF protection enabled
  - XFrame options configured
  - Session security enabled
  - OTP/2FA support enabled
  - Secure password validation

TROUBLESHOOTING
================================================================================
Common Issues:

1. ModuleNotFoundError - Install missing dependencies:
   pip install -r requirements.txt

2. Database migration errors - Apply migrations:
   python manage.py migrate

3. Static files not loading - Collect static files:
   python manage.py collectstatic --noinput

4. Port already in use - Run on different port:
   python manage.py runserver 8001

5. Permission denied on Linux - Activate virtual environment:
   source venv/bin/activate
